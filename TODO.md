## Project: ohmyqwen — QMD 기반 하이브리드 검색 통합 (폐쇄망/로컬 임베딩 전제)

너는 시니어 엔지니어다.
현재 코드베이스를 먼저 읽고, **기존 구조를 최대한 존중**하며 최소 변경으로 기능을 확장해라.

---

## 0) 미션

ohmyqwen의 에이전틱 코딩 루프 정확도를 높이기 위해:

1. Retrieval를 교체 가능한 구조로 정리
2. QMD를 1순위 검색으로 통합
3. 로컬 임베딩 서비스(폐쇄망) 전제 지원
4. 하이브리드 검색 + 토큰 예산 패킹 결합
5. Verify 실패 피드백을 검색에 재주입
6. 인덱스 수명주기(증분/스테일 감지) 구축
7. 평가 하네스 + 운영 문서 완성

---

## 1) 핵심 원칙

- **Rewrite 금지**: 기존 런타임/루프를 살리고 확장해라.
- **Minimal invasive change**: 필요한 만큼만 수정.
- **Offline-first**: 프로그램 실행 중 외부 네트워크 의존 전제 금지.
- **Offline-install**: 구현 완료 되면, 의존성 통합 묶음 패키지로 폐쇄망에 반입 실행.
- **Graceful degrade**: QMD/임베딩 장애 시 lexical-only로 자동 강등.
- **Deterministic control**: 런타임 제어권은 모델이 아니라 시스템이 가진다.
- **Observability first**: 왜 실패했는지 로그로 설명 가능해야 한다.

## 2) 실행 프로토콜 (반드시 준수)

1. 먼저 코드베이스 스캔 후 **현재 구조 요약** 출력
2. 이번 작업의 **구현 계획(작업 쪼개기)** 출력
3. 구현
4. 검증 실행
5. 결과/리스크/TODO 보고
6. 커밋

작업 도중 요구사항 충돌 시:
- 기존 설계를 우선 존중하고,
- 왜 그렇게 선택했는지 근거를 기록해라.

---

## 3) 구현 범위 (큰 그림 기준)

### A. Retrieval 추상화
- provider 인터페이스/타입/팩토리/체인 구조 도입
- 검색 엔진 교체가 쉬운 구조로 정리
- 기존 packer/runner와 자연스럽게 연결

### B. QMD 우선 + Fallback
- QMD provider를 primary로 연결
- 실패/타임아웃/파싱 오류 시 lexical fallback 자동 전환
- run 전체는 중단하지 않고 계속 진행

### C. 로컬 임베딩 연동 (폐쇄망)
- 로컬 임베딩 endpoint 설정 지원
- 시작 preflight에서 헬스체크
- 임베딩 서버 다운 시 semantic 비활성 + lexical-only 강등

### D. 하이브리드 검색 + 패킹
- lexical + semantic 결과를 통합/중복제거/rerank
- 입력 신호: task + 최근 diff + verify 오류
- 패킹 우선순위:
- 타겟 원문
- 오류 근거
- 최근 변경
- 요약
- stage별 토큰 하드캡 적용 (토큰 사이즈 설정 가능하도록 구성.)

### E. Verify 피드백 루프
- 실패 시그니처 추출
- PATCH 전 검색 쿼리에 실패 신호 반영
- 동일 실패 반복 시 전략 전환 또는 안전 종료(아티팩트 남김)

### F. 인덱스 라이프사이클
- 증분 갱신(변경 파일 중심)
- stale index 감지(모델/청크/설정 변화)
- 재인덱싱 유도 경로 + doctor 성격 진단 경로

### G. 평가 하네스 & 문서
- 대표 태스크셋 기반 평가 스크립트
- 지표:
- run 성공률
- 평균 loop 횟수
- verify 1회 통과율
- retrieval 품질 지표(적절한 방식 선택)
- 운영 문서:
- retrieval 구조
- 오프라인 임베딩 운영/장애대응
- 실행 가이드
